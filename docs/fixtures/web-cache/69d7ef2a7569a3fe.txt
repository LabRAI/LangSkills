<p><i>Core utilities</i> are the basic, fundamental tools of a <a href="https://en.wikipedia.org/wiki/GNU/Linux_naming_controversy" class="extiw" title="wikipedia:GNU/Linux naming controversy">GNU/Linux</a> system. This article provides an incomplete overview of them, links their documentation and describes useful alternatives. The scope of this article includes, but is not limited to, the <a href="https://en.wikipedia.org/wiki/GNU_Core_Utilities" class="extiw" title="wikipedia:GNU Core Utilities">GNU Core Utilities</a>. Most core utilities are traditional <a href="https://en.wikipedia.org/wiki/Unix" class="extiw" title="wikipedia:Unix">Unix</a> tools and many were standardized by <a href="https://en.wikipedia.org/wiki/POSIX" class="extiw" title="wikipedia:POSIX">POSIX</a> but have been developed further to provide more features.
</p><p>Most command-line interfaces are documented in <a href="/title/Man_page" title="Man page">man pages</a>, utilities by the <a href="/title/GNU_Project" class="mw-redirect" title="GNU Project">GNU Project</a> are documented primarily in <a href="/title/Info_manual" class="mw-redirect" title="Info manual">Info manuals</a>, some <a href="/title/Shell" class="mw-redirect" title="Shell">shells</a> provide a <i>help</i> command for shell builtin commands. Additionally most utilities print their usage when run with the <code>--help</code> flag.
</p>

<div class="mw-heading mw-heading3"><h3 id="Preventing_data_loss">Preventing data loss</h3></div>
<p><code>rm</code>, <code>mv</code>, <code>cp</code> and shell redirections happily delete or overwrite files without asking. <code>rm</code>, <code>mv</code>, and <code>cp</code> all support the <code>-i</code> flag to prompt the user before every removal / overwrite. Some users like to enable the <code>-i</code> flag by default using <a href="/title/Alias" class="mw-redirect" title="Alias">aliases</a>. Relying upon these shell options can be dangerous, because you get used to them, resulting in potential data loss when you use another system or user that does not have them. The best way to prevent data loss is to create <a href="/title/Backup" class="mw-redirect" title="Backup">backups</a>.
</p>

<div class="mw-heading mw-heading4"><h4 id="Alternative_dd_implementations">Alternative dd implementations</h4></div>
<p>This subsection lists <i>dd</i> implementations whose interface and default behaviour is mostly compliant with the POSIX specification of <span class="plainlinks archwiki-template-man" title="$ man 1p dd"><a rel="nofollow" class="external text" href="https://man.archlinux.org/man/dd.1p">dd(1p)</a></span>.
</p>
<ul><li><b>ddpt</b> &#8212; A portable rewrite of <span class="plainlinks archwiki-template-man" title="$ man 8 sg&#95;dd"><a rel="nofollow" class="external text" href="https://man.archlinux.org/man/sg_dd.8">sg_dd(8)</a></span> by the SCSI subsystem maintainer of the Linux kernel, with optional but very specialised hardware I/O (SCSI command sets) support, plus many other features.</li></ul>
<dl><dd><a rel="nofollow" class="external free" href="https://sg.danny.cz/sg/ddpt.html">https://sg.danny.cz/sg/ddpt.html</a> || <span class="plainlinks archwiki-template-pkg"><a rel="nofollow" class="external text" href="https://aur.archlinux.org/packages/ddpt/">ddpt</a></span><sup><small>AUR</small></sup></dd></dl>
<ul><li><b>sdd</b> &#8212; A dd implementation portable across UNIX environments by Joerg Schilling, that can checksum the copied data and retry reading bad blocks.</li></ul>
<dl><dd><a rel="nofollow" class="external free" href="https://schilytools.sourceforge.net/">https://schilytools.sourceforge.net/</a> || <span class="plainlinks archwiki-template-pkg"><a rel="nofollow" class="external text" href="https://aur.archlinux.org/packages/schily-tools-sdd/">schily-tools-sdd</a></span><sup><small>AUR</small></sup></dd></dl>
<div class="mw-heading mw-heading5"><h5 id="Spin-offs_of_GNU_dd">Spin-offs of GNU dd</h5></div>
<p>The GNU implementation of <i>dd</i> found in <span class="plainlinks archwiki-template-pkg"><a rel="nofollow" class="external text" href="https://archlinux.org/packages/?name=coreutils">coreutils</a></span> also conforms to POSIX. This subsection lists its forks.
</p>
<ul><li><b><a href="https://en.wikipedia.org/wiki/Dd_(Unix)#dc3dd" class="extiw" title="wikipedia:Dd (Unix)">dc3dd</a></b> &#8212; Another patched version of GNU dd from the United States Department of Defense Cyber Crime Center (DC3), with similar goals and features to dcfldd.</li></ul>
<dl><dd><a rel="nofollow" class="external free" href="https://sourceforge.net/projects/dc3dd/">https://sourceforge.net/projects/dc3dd/</a> || <span class="plainlinks archwiki-template-pkg"><a rel="nofollow" class="external text" href="https://aur.archlinux.org/packages/dc3dd/">dc3dd</a></span><sup><small>AUR</small></sup></dd></dl>
<ul><li><b><a href="https://en.wikipedia.org/wiki/Dd_(Unix)#dcfldd" class="extiw" title="wikipedia:Dd (Unix)">dcfldd</a></b> &#8212;  feature-enhanced fork of GNU dd for forensics and security scenarios, includes on-the-fly hashing capability, flexible wipes, write verification, output to multiple targets at the same time, split and piped output.</li></ul>
<dl><dd><a rel="nofollow" class="external free" href="https://dcfldd.sourceforge.net">https://dcfldd.sourceforge.net</a> || <span class="plainlinks archwiki-template-pkg"><a rel="nofollow" class="external text" href="https://aur.archlinux.org/packages/dcfldd/">dcfldd</a></span><sup><small>AUR</small></sup></dd></dl>

<div class="mw-heading mw-heading2"><h2 id="Tips_and_tricks">Tips and tricks</h2></div>
<div class="mw-heading mw-heading3"><h3 id="Override_or_add_missing_coreutils">Override or add missing coreutils</h3></div>
<p>Some commands (<code>arch</code>, <code>kill</code>, etc.) are missing from <span class="plainlinks archwiki-template-pkg"><a rel="nofollow" class="external text" href="https://archlinux.org/packages/?name=coreutils">coreutils</a></span> or taken from other packages. To complete them for compatibility, install <span class="plainlinks archwiki-template-pkg"><a rel="nofollow" class="external text" href="https://archlinux.org/packages/?name=uutils-coreutils">uutils-coreutils</a></span> and do:
</p>
<pre># ln -sf /usr/bin/uu-coreutils /usr/local/bin/arch
# echo -e "#compdef arch=uu-arch\n_uu-arch" &gt; /usr/local/share/zsh/site-functions/_arch
# echo "complete -c arch -w uu-arch" &gt; /usr/local/share/fish/vendor_completions.d/arch.fish
</pre>
