# NSF 风格项目申请书：合规清单 + 写作大纲（PAPPG）

## Goal
- 以 NSF 为代表，建立一套“先合规、再叙事、后打磨”的写作流程：用 checklist 保证格式与材料齐全，用大纲保证研究故事清晰可评审。

## When to use
- 你要写 NSF 或其他相似结构的科研基金申请，需要把要求拆成可执行步骤并避免合规踩雷。
- 你需要一个可复用的大纲模板，用于团队协作与内部评审（PI/Co-PI/OSP）。

## When NOT to use
- 资助机构是完全不同结构（例如特定基金的专用表格与章节），应以其官方模板为准。
- 你打算跳过机构内部流程或在预算/承诺上夸大不实（高风险且可能违规）。

## Prerequisites
- Environment: 文档编辑器 + 机构提交系统（通常需 OSP/Research Office 流程）
- Permissions: PI/Co-PI 信息、机构资质、预算相关权限（通常需 OSP 协助）
- Tools: checklist（表格）+ 版本管理（便于多轮评审）
- Inputs needed: solicitation/program 说明、截止日期、PAPPG/机构政策、团队分工、预算约束

## Steps (<= 12)
1. 锁定 solicitation/program 与截止日期，通读 “preparing proposal” 指南，列出必须提交的材料清单与版本要求。[[1][2]]
2. 用 PAPPG Chapter II 建一份合规 checklist：页数/字体/边距、章节结构、required supplementary docs、命名与上传规则。[[2]]
3. 先写 1 页 Project Summary：Intellectual Merit + Broader Impacts + 目标与关键创新点（便于后续对齐全文）。[[2][1]]
4. 按 PAPPG 结构搭 Project Description 大纲：背景与动机 → 相关工作与缺口 → 研究目标 → 方法与工作包 → 预期结果 → 评估/里程碑/时间线。[[2]]
5. 把 Broader Impacts 与 dissemination/data sharing 计划写成可执行条目，并与学校资源/合作方对齐。[[2][1]]
6. 明确管理与分工：PI/Co-PI 角色、学生培养、风险与备选方案；若需要 postdoc mentoring plan 等附录，提前准备。[[2]]
7. 与 OSP 共同做预算与预算说明（justification）：每项预算都能对应工作包与交付物，并符合允许范围。[[1][3]]
8. 做一次内部红队评审：让 1 位“非本方向”读者只看 summary+大纲，看能否讲清价值与可行性；再让 1 位“懂行”读者挑漏洞与风险。[[3][1]]
9. 最后合规回归：对照 checklist 抽检页数、附件、格式、引用一致性；留出机构内部审批与系统故障缓冲时间。[[1][3]]

## Verification
- Checklist 全部打勾：格式/页数/章节/附件齐全且命名符合要求。
- Summary 与全文一致：目标、创新点、工作包、broader impacts 不冲突。
- 最终 PDF 可读且图表清晰；引用/交叉引用无断链。

## Safety & Risk
- Risk level: **medium**
- Irreversible actions: 错误提交/不合规可能导致直接拒收；预算与承诺不实可能触发合规与信誉风险
- Privacy/credential handling: 预算、工资比例、合作信息可能敏感；内部评审材料按机构要求保密
- Confirmation requirement: 在提交前必须完成 OSP/机构流程复核；所有预算与承诺需可被审计与执行
