# Safety Policy（必须遵守）

本仓库收录的是“可执行的任务流程”，安全边界必须写死在仓库里。

## 禁止收录

- 绕过风控/欺诈/盗号/钓鱼/灰产
- 未授权渗透、漏洞利用与入侵指导
- 制造武器/爆炸物/有毒物质等危险内容
- 任何需要用户违法或侵犯他人权益的流程

## 高风险操作最低要求

以下类型默认视为高风险：删除数据、权限变更、支付/提交、影响生产系统的变更。

skill 必须包含：

- 明确警告（会影响什么、不可逆点是什么）
- 安全预演步骤（dry-run / echo / show plan / 只读检查）
- “明确确认”的语言（例如：`Only run this if ...`）
- Verification（如何确认没有误操作）

## 凭证与隐私

- 不要让用户把 token/密码写进仓库或日志
- 示例中用占位符；提示使用环境变量或安全存储
